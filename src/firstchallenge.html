Challenge #1 - Quicksort

Introduction

Quicksort is a sorting algorithm that is meant to be faster and more efficent than insertion sort or bubble sort. For this challenge, try and implement quicksort to sort a list of numbers of any size.

The Challenge Details

Quicksort involves the following:

1) Start with the whole list

2) Pick a random value from the list - it does not have to be from the middle. This will be our pivot.

3) Reorder the list, moving (in no particular order) everything that is smaller than the pivot to the left of it and everything that's greater than the pivot to the right of it.  Now, the list to the left of, not including, the pivot is known as list S, and the list to the right of, not including, the pivot is known as list G.  S and G don't have to be created in order, just so long as they are all smaller or greater than the pivot respectively.

4) Now repeat step 2 onward for lists S and G. If the list only contains zero or one items, you can stop, as it's by default sorted. If either only contains 2 items, it might make it quicker to just compare and swap if necessary instead of doing the whole sorting procedure.

Main objective of Challenge

Using the knowledge provided above and doing a quick google search on more info about quicksort, quicksort a long list of arbitarary numbers using your own method of quicksort, not a built-in library method.

Allowed Inputs and Outputs

Ask the user for the size of the list, and then ask the user to fill in the list according to the size. Then, quicksort the list of numbers (including decimals).
